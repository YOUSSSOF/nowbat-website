# Page Wireframes (Low-Fidelity)

> Phase 2 output. These wireframes lock in the layout structure of every key page. Dimensions are relative — no pixel precision at this stage. RTL layout is assumed throughout (right-to-left reading order).

---

## Layout Conventions

```
[ ]  = GlassCard or content block
███  = Image / screenshot / mockup
---  = Horizontal divider or border
|||  = Vertical divider
RTL: content flows right → left. "Left" in these wireframes = leading edge (right in browser)
```

## Mobile Layout Behavior

| Section                | Mobile Behavior                                                                        |
| ---------------------- | -------------------------------------------------------------------------------------- |
| `HeroSection`          | Columns stack: text first, screenshot second. Screenshot is 90vw wide, capped at 480px |
| `AlternatingFeature`   | Always stacks vertically. `reverse` prop has no effect below `lg` breakpoint           |
| `FeatureGrid`          | 1 column on mobile, 2 on `sm`, 3 on `lg`                                               |
| `StatStrip`            | 2×2 grid on mobile                                                                     |
| `HowItWorksSection`    | Vertical stack, no connecting line on mobile                                           |
| `TestimonialsCarousel` | Shows 1 card at a time, swipeable                                                      |
| `DocsSidebar`          | Hidden, available via drawer button                                                    |
| `OnThisPage`           | Hidden, available via sticky button                                                    |
| `SiteHeader`           | Hamburger menu replaces nav links                                                      |
