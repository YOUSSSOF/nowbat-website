# نصب افزونه نوبت

## پیش‌نیازها

قبل از نصب، مطمئن شوید محیط شما شرایط زیر را دارد:

| پیش‌نیاز | حداقل نسخه |
|---|---|
| WordPress | ۶.۰ |
| PHP | ۸.۱ |
| MySQL | ۸.۰ یا MariaDB ۱۰.۶ |

افزونه نوبت به خروجی‌های PHP اضافی نیاز ندارد فراتر از آنچه وردپرس به آن نیاز دارد. اگر سایت وردپرسی شما اکنون کار می‌کند، احتمالاً نوبت هم نصب می‌شود.

## روش ۱ — نصب از پیشخوان وردپرس

۱. وارد پیشخوان وردپرس خود شوید.
۲. از منو **افزونه‌ها ← افزودن افزونه جدید** را انتخاب کنید.
۳. در کادر جستجو عبارت **نوبت** یا **Nowbat** را تایپ کنید.
۴. روی **نصب** کلیک کنید، سپس **فعال‌سازی**.

پس از فعال‌سازی، بلافاصله ویزارد اولیه باز می‌شود.

## روش ۲ — آپلود فایل ZIP

پس از خرید از [راستچین](https://www.rtl-theme.com/product/nowbat/)، فایل ZIP را دانلود کرده و از طریق پیشخوان نصب کنید:

۱. **افزونه‌ها ← افزودن افزونه جدید ← بارگذاری افزونه**
۲. فایل `nowbat-x.x.x.zip` را انتخاب کنید.
۳. روی **نصب** و سپس **فعال‌سازی** کلیک کنید.

## روش ۳ — WP-CLI

```bash
# آپلود از فایل ZIP خریداری‌شده
wp plugin install /path/to/nowbat.zip --activate
```

## ویزارد اولیه

پس از فعال‌سازی، وردپرس شما را به **تنظیمات ← نوبت** هدایت می‌کند. ویزارد چهار مرحله دارد:

1. **اطلاعات کسب‌وکار** — نام، منطقه زمانی، فرمت تاریخ
2. **درگاه پرداخت** — انتخاب یک درگاه پیش‌فرض (زرین‌پال، آیدی‌پی، یا نکست‌پی)
3. **سرویس پیامک** — انتخاب ارائه‌دهنده و وارد کردن API key
4. **ساعت کاری** — ساعت‌های کاری پیش‌فرض برای همه کارمندان

می‌توانید هر مرحله را رد کنید؛ تنظیمات بعداً از **نوبت ← تنظیمات** قابل ویرایش است.

## تأیید نصب

برای اطمینان از نصب صحیح:

```bash
wp plugin list | grep nowbat
```

باید خروجی زیر را ببینید:

```
nowbat    active    x.x.x
```

همچنین جداول پایگاه‌داده باید ایجاد شده باشند:

```bash
wp db query "SHOW TABLES LIKE '%nowbat%';" --skip-column-names
```

<Callout type="info">
اگر جداول ایجاد نشدند، افزونه را غیرفعال کرده و دوباره فعال کنید. وردپرس هنگام فعال‌سازی hook `register_activation_hook` را اجرا می‌کند که مسئول ایجاد جداول است.
</Callout>

## حذف نصب

برای حذف کامل:

```bash
wp plugin deactivate nowbat
wp plugin delete nowbat
```

داده‌های پایگاه‌داده و تنظیمات **حذف نمی‌شوند** مگر اینکه از **نوبت ← تنظیمات ← پیشرفته** گزینه «حذف داده‌ها هنگام حذف افزونه» را فعال کرده باشید.
